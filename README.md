browser-proctoring-recorder

ブラウザのみで動作する、オンライン試験・テスト監視（プロクタリング）用の同時録画アプリケーションです。学生のインカメラ映像とディスプレイ（画面全体）の映像を同時に録画し、それぞれの動画ファイルの改ざん防止用ハッシュ値（SHA-256）を計算して、1つのZIPファイルにまとめて出力します。

完全なクライアントサイド（ブラウザ上）処理のため、サーバーへの動画アップロード負荷やプライバシーへの懸念がありません。

🌟 主な機能

📷 デュアル録画: インカメラ（映像・音声）とディスプレイ（画面共有）の同時録画

🔒 整合性検証 (Hashing): Web Crypto APIを使用した高速なSHA-256ハッシュ値の自動計算

📦 ワンパック出力: 2つの動画ファイルとハッシュ値が記載された metadata.json を1つのZIPファイルにまとめてダウンロード

☁️ サーバーレス: すべての処理を学生のブラウザ内で行うため、プライバシーに優しくサーバー費用も不要

🚀 使い方 / 導入方法

1. ローカルで実行する場合

本リポジトリをクローンまたはダウンロードします。

index.html をブラウザ（Chrome / Edgeなど）で開きます。

※注意: ブラウザのセキュリティ仕様により、カメラや画面共有のAPIは HTTPS環境 または localhost でのみ動作します。

2. GitHub Pages で公開する場合

このリポジトリを自身のGitHubアカウントにフォーク/作成します。

リポジトリの Settings > Pages に移動します。

Build and deployment の Source で「Deploy from a branch」を選択し、main（または master）ブランチの / (root) を設定して保存します。

数分後、自動的に生成される https://<あなたのユーザー名>.github.io/<リポジトリ名>/ のURLからアクセス可能になります。

録画の手順

「録画開始 (Start)」 ボタンをクリックします。

ブラウザからカメラとマイクのアクセス権限を求められるので「許可」します。

画面共有の選択画面が表示されるので、「画面全体 (Entire Screen)」 を選択して共有します。

試験終了後、「録画停止・ZIP保存 (Stop & Save)」 ボタンをクリックします。

自動的にハッシュ値が計算され、動画とメタデータが含まれた .zip ファイルがダウンロードされます。

📂 ZIPファイルに含まれる内容

ダウンロードされるZIPファイル内は以下の構成になっています。

exam_record_[タイムスタンプ].zip
├── camera_video.webm      # インカメラの録画データ（音声付き）
├── screen_video.webm      # ディスプレイ（画面全体）の録画データ
└── metadata.json          # 録画日時と各動画のSHA-256ハッシュ値


metadata.json の出力例

{
  "generated_at": "2026-07-07T03:30:00.000Z",
  "files": {
    "camera_video.webm": {
      "sha256": "8f43c0a1a... (カメラ動画のハッシュ値)",
      "size_bytes": 102400
    },
    "screen_video.webm": {
      "sha256": "a3f12b9c8... (画面動画のハッシュ値)",
      "size_bytes": 204800
    }
  }
}


🛠️ 使用している技術・ライブラリ

MediaStream Recording API (MediaRecorder): ブラウザ上でのリアルタイム動画エンコード

Screen Capture API (getDisplayMedia): ディスプレイ映像の取得

Web Crypto API (crypto.subtle.digest): ブラウザ側での高速なSHA-256ハッシュ計算

JSZip: JavaScriptによるクライアントサイドでのZIP圧縮処理 (CDN経由)

⚠️ 運用上の注意点

対応ブラウザ: PC版の Google Chrome, Microsoft Edge などの Chromium 系ブラウザ、および Firefox で動作します。スマートフォン（iOS/Android）ではブラウザの仕様上、画面共有（getDisplayMedia）がサポートされていないため動作しません。

PCの負荷: 2つの動画を同時にエンコードし、最後にハッシュ計算を行うため、学生のPCスペックによっては動作が重くなる場合があります。

不正防止の限界: 本ツールは完全なクライアントサイドで動作するデモ・骨子です。実際のオンライン試験で厳格に運用する場合は、ZIPファイルを自動的に試験管理サーバーへ非同期送信（APIアップロード）する仕組みへの拡張を推奨します。
